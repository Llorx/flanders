# `implement` Command Contract — Overview

## Purpose
Execute a plan file end-to-end: implement each open task with a worker AI, validate the result through build, test, and adversarial-review gates, and mark each accepted task as complete in the plan file.

## Invocation
    npx flanders implement [plan] [--no-git]

- When `[plan]` is given, it is treated as the path to a plan markdown file under the `plans/` folder.
- When `[plan]` is omitted:
  - If `plans/` contains exactly one file, that file is selected automatically.
  - If `plans/` contains more than one file, the command interactively prompts the user to choose one.
  - If `plans/` contains no files, the command exits with an error.
- `--no-git` disables the git integration for the run even when git is available and the current directory is inside a git working tree. See `cli-commands/implement/git-integration.md`.

## High-level flow
1. Read the selected plan file.
2. Validate the plan file's format programmatically before doing anything else: every leaf task line must follow the shape defined in `shared/plan-file-format.md`, including a valid checkbox (no malformed variants such as `[]`, `[ x]`, `[X ]`) and a metrics object of the form `{"it":N,"ot":N,"t":N}` that is parseable as strict JSON with non-negative integer values for all three fields. At least one task line must be detected. On any validation failure, exit non-zero with a diagnostic that names the problem, the path to the offending file, and the offending line. This validation runs unconditionally regardless of how the plan was produced, because plan files generated by `/flanders-plan` are no longer validated by a wrapper at generation time.
3. Detect which tasks are open and which are completed using the rules from `shared/plan-file-format.md`.
4. If every task is already marked complete, print `tasks completed` and exit successfully.
5. When the git integration is active (see `cli-commands/implement/git-integration.md`), run the git preflight check. If the preflight fails, exit non-zero before setting up any workspace.
6. Set up the run workspace (see `cli-commands/implement/workspace.md`).
7. Run the iteration loop (see `cli-commands/implement/iteration-loop.md`), advancing one task at a time.
8. Throughout the run, render the live UI defined in `cli-commands/implement/ui.md`.

## Outcomes
- Plan validation failure — exit non-zero at startup when the plan file is missing, empty, or contains malformed task lines. The diagnostic names the problem and the path to the file.
- Git preflight failure — exit non-zero at startup when the git integration is active and the working tree has uncommitted changes other than the selected plan file. See `cli-commands/implement/git-integration.md`.
- `tasks completed` — printed and exit successfully when every task in the plan was already marked complete at startup.
- `all tasks completed` — printed and exit successfully when every remaining task was implemented and accepted during this run.
- Hard stop — printed and exit non-zero when the per-task iteration cap is exceeded for any single task.
